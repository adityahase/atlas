Implement the UI solutions. Start with @ui/research/README.md  

The solutions are carefully written, but you might face issues during implementation. Follow the intent, adapt to what you find. Think deeply and promptly present the issues.

For each solution
- Pick an unimplemented solution
- Read all context files completely
- Run unit tests on atlas.tests.local if you change the Python code
- Using Playwright MCP, confirm the implementation matches taste and intent 
- Generate the wireframe of implementation
- If the executed solution doesn't match the plan and wireframe, then track it in a drift file
- Update the solution as implemented
- Record screenshots of the final solution in the ui/research/screenshots directory

At all costs
- Implement using standard Frappe Framework patterns. 
- Refer to frappe-dev skills for all framework-related changes. 
- Refer to PATTERNS from frappe-ui skill for UI patterns.
- Stick to Frappe and Espresso guidelines (llm/references/frappe-ui)
- Refer to frappe, erpnext, crm and gameplan repositories for patterns.
- Instead of multiple buttons. Keep all actions in one Actions dropdown.
- Keep one primary button on every page.
- Instead of an HTML component, use standard components.
- Use fewer colours and font styles, font-weight, etc.
- Use consistent line height, spacing and padding.
